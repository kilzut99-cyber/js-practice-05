# 📑 Практическая работа №5: BOM, Таймеры и Автоматизация (Уровень PRO)

## 🎯 Цель работы
Глубокое изучение и практическое применение **Browser Object Model (BOM)**. Основной упор сделан на управление асинхронными процессами во времени (`setTimeout`, `setInterval`), работу с системными диалогами и использование постоянного хранилища данных `localStorage` для создания отказоустойчивых веб-приложений.

---

## 🛠 Используемые технологии

### **1. HTML5 (Структура и Семантика)**
*   **Семантическая разметка:** Использование тегов `<main>`, `<section>`, `<header>` для обеспечения правильной структуры и доступности (Accessibility).
*   **Интерактивные элементы:** Внедрение `<input type="number">` с валидацией диапазонов, `<textarea>` для работы с текстом и динамических контейнеров для системных уведомлений.

### **2. CSS3 (Интерфейс и Темы)**
*   **Система тем (Theming):** Реализация светлой и темной тем через **CSS-переменные** (`:root`) и динамические классы. Темы переключаются как автоматически (по времени суток), так и принудительно (кнопки-иконки).
*   **Flexbox Layout:** Гибкая верстка карточек заданий и центрирование элементов управления для корректного отображения на разных экранах.
*   **BOM-интеграция:** Использование `fixed` позиционирования для элементов, привязанных к окну браузера (уведомления в углу, переключатели тем).
*   **Анимации:** Плавные переходы `transition` для смены цветовых схем и `@keyframes` для анимации появления уведомлений.

### **3. JavaScript (ES6+ и BOM API)**
*   **Asynchronous Timers:** Управление жизненным циклом приложения через `setInterval` (для цикличных процессов вроде секундомера) и `setTimeout` (для отложенных событий вроде скрытия плашек).
*   **BOM API Deep Dive:**
    *   `window.navigator`: Идентификация устройства пользователя (Mobile/Desktop) и мониторинг статуса сети (Online/Offline).
    *   `window.location`: Программная навигация, анализ текущего URL и домена.
    *   `window.localStorage`: Энергонезависимое сохранение состояния таймеров и черновиков между перезагрузками страницы.
*   **DOM Manipulation:** Использование исключительно `addEventListener` для соблюдения принципа разделения логики и разметки (отказ от `onclick`).

## ❓ Ответы на вопросы для интервью (по лекции и выполненной работе №5)

> **1. В чём разница между `setTimeout()` и `setInterval()`? Когда использовать каждый?**
*   **`setTimeout()`** планирует выполнение кода **один раз** через заданную паузу. Идеален для разовых акций (например, скрыть уведомление через 5 секунд).
*   **`setInterval()`** запускает циклическое выполнение кода через равные промежутки. Необходим для процессов, требующих постоянного обновления: часов, секундомеров и систем автосохранения.

> **2. Что произойдёт, если не вызвать `clearInterval()`? Какие проблемы это может вызвать?**
*   Функция внутри интервала продолжит выполняться вечно в фоновом режиме, даже если она больше не нужна. Это ведет к **утечкам памяти** (memory leaks), лишней нагрузке на процессор (CPU) и логическим ошибкам при повторных запусках того же процесса.

> **3. Что такое BOM? Назовите основные объекты BOM и их назначение.**
*   **BOM (Browser Object Model)** — это набор объектов, предоставляемых браузером для управления окружением вне страницы.
*   `window`: Глобальный объект, содержащий всё остальное.
*   `navigator`: Информация о браузере, ОС, языке и сетевом статусе.
*   `location`: Работа с текущим адресом (URL), возможность перенаправления.
*   `history`: Управление историей переходов вкладки.
*   `screen`: Данные о физическом мониторе пользователя (разрешение).

> **4. Как узнать, с мобильного ли устройства зашёл пользователь? Приведите пример кода.**
*   Браузер предоставляет строку `userAgent`, которую можно проверить регулярным выражением на наличие меток мобильных платформ.
*   **Код:** `const isMobile = /Android|iPhone|iPad|iPod/i.test(navigator.userAgent);`

> **5. Почему `alert()`, `confirm()` и `prompt()` считаются устаревшими? Какие у них недостатки?**
*   Они являются **блокирующими**: пока окно открыто, основной поток (Main Thread) исполнения JS замирает, анимации и таймеры могут работать некорректно. Также их невозможно стилизовать, что портит дизайн приложения.

> **6. Что делает метод `padStart()`? Зачем он нужен при работе с таймерами?**
*   Он дополняет строку в начале определенными символами до достижения заданной длины. 
*   **Зачем:** Это необходимо для поддержания формата времени (например, превратить число `5` в строку `"05"`), чтобы дисплей всегда отображал `00:05:01` вместо `0:5:1`.

> **7. Как сохранить состояние таймера в `localStorage`, чтобы при перезагрузке он продолжал работать?**
*   Нужно сохранять текущее количество секунд и метку времени старта (`Date.now()`). При загрузке страницы скрипт вычитает из текущего времени сохраненную метку и прибавляет разницу к счетчику, имитируя непрерывный ход таймера.

> **8. Что такое `location.href`? Как перенаправить пользователя на другую страницу?**
*   Это строковое свойство объекта `location`, содержащее полный URL текущей страницы. 
*   **Перенаправление:** Присвоение нового значения свойству: `window.location.href = 'https://google.com';` заставит браузер немедленно перейти по адресу.
